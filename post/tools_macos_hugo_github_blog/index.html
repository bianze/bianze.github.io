MacOS Hugo+github pages搭建个人博客

2020-06-27
实用工具

Contents

  • 前言
  • Hugo安装
  • Hugo使用
  • 创建GitHub blog仓库
  • Hugo与github进行关联

前言

一直以来都希望能有个自己的博客，作为自己职业生涯的记录。网上搜索了下，发现最近比较通用的方式是通过hugo+github pages来实现，首先简单介绍下hugo和github pages。

  • Hugo 是一个用 Go 语言编写的静态站点生成器，它针对速度、易用性和可配置性进行了优化，快速灵活。
  • GitHub Pages 是一个静态站点托管服务，直接将个人、组织或项目的页面托管于 GitHub 库或仓库 (repository) 中。

我本人平时用的是macbook，下面就以macOS来实践下，如何搭建个人博客。

Hugo安装

1.浏览hugo官方的安装指南

浏览指南，对安装过程有初步了解，macOS上安装hugo前先安装homebrew和go，homebrew是Mac上很常用的安装命令。

hugo安装指南：https://gohugo.io/getting-started/installing/

2.打开terminal，安装hugo
1

brew install hugo
3.查看hugo版本
1

hugo version

这里有个坑，后面的主题even对hugo的版本有要求，而mac这样直接安装的版本不符合要求，后来直接在hugo官网找到二进制下载解压后将二进制直接放到/usr/local/bin目录即可。

Hugo使用

1.创建hugo网站

进入想要存放hugo网站文件的目录，例如我的是

1

cd ~/Documents/

执行以下命令，新建一个hugo网站

1

hugo new site blog  # "blog" 是我的网站文件夹名。

创建好后，可以看到一个新的blog目录，进入blog目录可以看到有一些子目录和初始配置文件。

2.选择hugo主题

hugo有很多主题可供选择，可以在https://themes.gohugo.io 查看，我这里选择even, 直接clone到themes目录下

1
2
3

cd themes

git clone https://github.com/olOwOlo/hugo-theme-even.git even

用even下的示例配置文件替换初始配置文件

1
2
3

cd ..

cp themes/even/exampleSite/config.toml ./config.toml

然后可以修改这个配置文件中的某些字段，定制成自己的，这个可以后面根据页面的效果进行修改即可。

3.新建一篇文章

进入网站文件夹的根目录，执行

1

hugo new post/my-first-post.md  # "my-first-post.md" 是新建文章的文件名。

此时在content/post目录下可以看到新创建的文章，可以使用typora这种markdown编辑器打开新建的文章进行编辑。

注意如果要提交到远端仓库，发表该文章，把文件最头上的draft从true改为false, 否则发布的时候不会被包含进去，这样提交到远端也看不到。

4.本地预览网站效果

启动Hugo server

1

hugo server -D

在浏览器中打开http://localhost:1313进行预览，对效果不满意的话再次编辑即可。

创建GitHub blog仓库

1.注册一个github账号

登陆github按照指示填写相关信息并验证通过进行注册即可，如果已经有账号跳过此步。

2.新建一个GitHub repository

用注册好的账号登陆github，进入你的仓库，然后点击new进行创建，我的仓库取名为bianze.github.io

Hugo与github进行关联

1.构建hugo网站

在 Hugo 网站文件夹的根目录下，执行 hugo 命令来构建。

1

hugo  # 构建你的 Hugo 网站，默认将静态站点保存到 "public" 目录。

注意：Hugo 会将构建的网站内容默认保存至网站根目录的 public/ 文件夹中。

2.建立hugo本地git仓库

进入public目录，执行

1

git init

将git本地仓库关联到github的远程仓库

1

git remote add origin https://github.com/bianze/bianze.github.io.git

此时执行

1

git remote -v

可以查看是否关联成功，或者查看.git/config文件。

3.提交修改，推送到github

将本地的修改进行提交，写好commit信息，然后推送远端仓库即可。

1
2
3
4
5

git add .

git commit -s

git push -u origin master --force
4.查看博客效果

在浏览器中输入https://bianze.github.io/便可看到自己的博客了。
